## 2. 安装与启动

### 2.1 安装依赖

在项目根目录（含 `requirements.txt`、`apps/`、`scripts/`）下执行：

```bash
pip install -r requirements.txt
```

### 2.2 配置可选文本与视觉分析 API（`.env`）

全自动脚本（`run_generate_eval_advise.py` 为主入口；`run_full_report.py` 为其便捷包装器）可在评审结束后，将 **指标摘要与统计** 发送至第三方 **文本生成 API**（`ADVISOR_TEXT_BACKEND=auto` 时优先级：DeepSeek > GLM（智谱）> DashScope Qwen），并在 `both` 模式下调用 **DashScope Qwen‑VL** 进行图像侧分析。实现见 `scripts/assistant/ask_advisor.py`。

| 用途 | 环境变量示例 | 说明 |
|------|----------------|------|
| 文本（默认 DeepSeek） | `DEEPSEEK_API_KEY` | 配置后优先使用；`ADVISOR_TEXT_BACKEND=qwen` 可强制仅走 Qwen 文本 |
| 文本（GLM 智谱） | `GLM_API_KEY`、`GLM_BASE_URL`、`GLM_MODEL` | 智谱 API；auto 模式下优先级高于 Qwen |
| 文本（仅 Qwen） | `QWEN_API_KEY`、`QWEN_BASE_URL`、`QWEN_MODEL` | DashScope OpenAI 兼容接口 |
| 看图 | `QWEN_API_KEY`、`QWEN_VL_MODEL` | 仅在 **`--advisor-mode both`** 时需要 |

```bash
cp .env.example .env
# 编辑 .env，按需填写上述变量（密钥勿提交 Git）
```

未配置密钥时：**图像生成与 `review_generated_images.py` 评审不受影响**；仅「自动生成 Markdown 建议」与「解析下一轮 JSON 参数」步骤会跳过或报错（界面与日志会提示）。

### 2.3 一键启动 Gradio + FastAPI（推荐）

```bash
bash apps/start.sh
```

| 服务 | 默认地址 |
|------|----------|
| Gradio | `http://127.0.0.1:7860` |
| FastAPI · Swagger | `http://127.0.0.1:8000/docs` |

**说明**：

- 前台运行，**`Ctrl+C`** 或执行 **`bash apps/stop.sh`** 可结束进程；**云主机 / AutoDL** 须在控制台做 **端口映射**，或通过 **Cursor「端口」转发**，否则在本机浏览器直接打开容器内的 `127.0.0.1` 会连不上。
- **`start.sh` 默认关闭** uvicorn **`--reload`**（与 Gradio 同仓时热重载易导致 **8000 被占用、FastAPI 子进程退出**）。开发时需要 API 热重载：**`bash apps/start.sh --api-reload`**。
- 画廊预览会在项目根目录生成 **`.gradio_gallery_cache/`**（已 `.gitignore`）；可用环境变量 **`GALLERY_PREVIEW_MAX`**、**`GALLERY_THUMB_MAX`**、**`GALLERY_DENY_PREFIXES`** 控制体量与隐藏批次前缀（详见 `outputs/generated/README.md`）。

改端口、`--gradio-share`、仅启一侧等：`bash apps/start.sh --help`（见脚本头部注释）。

### 2.4 单独启动（调试）

仅需界面或仅需 API 时再使用：

```bash
# 仅 Gradio
python3 apps/app_gradio.py --host 0.0.0.0 --port 7860

# 仅 FastAPI（是否加 --reload 自行决定；与 Gradio 同时开发时慎用 --reload）
uvicorn apps.api_server:app --host 0.0.0.0 --port 8000
```

## 3. Gradio 界面操作说明

当前顶栏共 **5 个标签页**（顺序）：**生成** → **画廊** → **评估** → **一键流水线** → **调参历史**。  
其中 **「生成」** 为答辩简化页：只保留出图所需的常用参数，高风险研究参数改由 CLI/API 使用。

### 3.1 环境与路径自检

打开 **「生成」** 标签页，滚动到底部展开 **「环境检查（模型 / 数据路径）」**，确认表格中 SD1.5 基座、LoRA、元数据 CSV 等为 ✅。可点 **「刷新状态」**；全局刷新多个面板时点 **「全局刷新」**。若有 ❌，按 README 核对路径。

### 3.2 生成（日常出图）

适用于：答辩演示和日常批量出图。界面固定使用 **全图模式**，不展示 Patch 拼接、LoRA 路径、LoRA scale 或 Negative Prompt。

| 区域 | 说明 |
|------|------|
| **速度预设** | **快速** / **精细**：自动写入采样步数、strength、CFG。需要时可展开 **高级采样参数** 微调这三项。 |
| **数据与输出** | 生成数量、体位 / 密度筛选、输出前缀、生成种子、源图种子（0=随机）。 |
| **常用开关** | 频域后处理默认开启；保存长边默认 1024，便于 Gradio 预览与答辩展示。 |
| **内部默认** | LoRA 固定为 `outputs/lora/mammo_sd15_v6_allMLO/final_lora`；负向提示词不在前端暴露。 |

点击 **「开始生成」**，完成后到 **「画廊」** 点 **「刷新」** 或直接查看自动选中的最新批次。

**参考默认值（精细预设）**

| 参数 | 默认值（标准预设） |
|------|---------------------|
| 采样步数 | 50 |
| Strength | 0.36 |
| Guidance Scale | 6.5 |
| Scheduler | dpm |

> **耗时**：与张数、步数和显卡有关；batch 较大时数分钟级属常见。

### 3.4 评估图像质量

切换到 **「评估」** 标签页：

1. 从下拉列表选择已生成的批次目录
2. 设置 Top-K（取质量最高的前 K 张），推荐 30
3. 评审模式选「full」（完整 16 维评审）
4. 点击 **「开始评估」**

评审结果保存至 `outputs/eval/<批次名>/`：
- `summary.json`：汇总指标
- `review_report.csv`：每张图详细评分
- `top_<K>_images/`：推荐的最优图

### 3.5 一键流水线

切换到 **「一键流水线」** 标签页：

1. 设置生成数量与输出 Tag 前缀（用于区分批次）
2. 选择速度预设、体位、密度、生成数量与随机种子；流水线同样固定走全图模式
3. 勾选频域后处理；LoRA 路径、LoRA scale 与 Negative Prompt 不在流水线面板展示
4. 点击 **「启动流水线」**

全流程结束后会更新 **`outputs/reports/LATEST_NEXT_RUN.json`**（若本轮产出顾问 JSON）。单独运行 **`run_generate_eval_advise.py`** 现在也会同步该文件，不再必须经过 `run_full_report.py`；CLI 可用 **`run_full_report.py --from-latest-tuning`** 复用推荐参数。

完成后：
- 评审报告在 `outputs/eval/`
- 可读性分析报告在 `outputs/reports/<tag>/advisor_suggestions.md`（`both` 时可能另有 `advisor_qwen_text.md` 等）
- 完整报告在 `outputs/reports/<tag>/FINAL_REPORT.md`
- 最新报告路径记录在 `outputs/reports/LATEST_REPORT.txt`

### 3.6 调参历史与回滚

切换到 **「调参历史」** 标签页：

- 点击 **「刷新历史」** 查看最近轮次的参数与指标表（数据来自 `PARAM_HISTORY.json`）
- 设置 **「回滚到第 N 条」**（1=最新）后点 **「执行回滚」**（将再次调用脚本，需配置 API 时使用）

### 3.7 画廊

切换到 **「画廊」** 标签页：

- **批次来源**：默认与 **「最新一批」** 对齐；历史批次可通过折叠区 **「路径选择」** 内的 **FileExplorer**（根目录一般为论文输出根）或 **手动相对路径** 指定目录后再加载（不会对 `outputs/generated` 整树扫描，避免卡顿）。
- **预览图**：界面使用 **JPEG 缩略图缓存**（`.gradio_gallery_cache/`），大图仅在选中路径后按需生成；可调 **`GALLERY_PREVIEW_MAX`** / **`GALLERY_THUMB_MAX`**。
- **评估汇总表**：进入画廊时表格 **默认为空**，需点击 **「刷新评估与批次列表」** 才读取各批次 `summary.json`（避免一打开就加载大量 JSON）。
- **旧输出归档**：若历史批次过多，可使用 `python3 scripts/tools/archive_legacy_generated_for_gallery.py`（建议先 `--dry-run`），详见 `outputs/generated/README.md`。

## 4. 命令行操作说明

### 4.1 生成图像

```bash
python3 scripts/generation/run_mammo_sd15.py \
  --base-model-local hf_cache/sd15 \
  --lora-path outputs/lora/mammo_sd15_v4_clean/final_lora \
  --metadata-csv datasets/CBIS_CLEAN_V2/metadata_clean.csv \
  --filter-view MLO --filter-density dense \
  --num-images 6 --seed 2026 \
  --strength 0.42 --guidance-scale 7.9 \
  --overlap-ratio 0.85 --gabor-alpha 0.50 \
  --global-guide-blend 0.44 --global-guide-strength 0.25 \
  --scheduler dpm --num-steps 20 --patch-size 640 \
  --output-subdir-prefix my_batch
```

### 4.2 评审图像

```bash
python3 scripts/evaluation/review_generated_images.py \
  --images-dir outputs/generated/my_batch_20260503_000000_000 \
  --output-dir outputs/eval/my_batch \
  --no-recursive --eval-profile full \
  --enable-seam-check \
  --real-images-dir datasets/jpeg   # 可选：真实图库路径，写入域内分项基线并可计算 academic_metrics（FID/KID/PRC/sFID）
```

### 4.3 多轮对比

```bash
python3 scripts/evaluation/compare_runs.py --runs-dir outputs/eval
```

### 4.4 全自动报告

```bash
# 主入口：生成 + 评审 + 顾问分析
python3 scripts/assistant/run_generate_eval_advise.py \
  --metadata-csv datasets/CBIS_CLEAN_V2/metadata_clean.csv \
  --filter-view MLO --filter-density dense \
  --num-images 6 --tag-prefix round10 \
  --advisor-mode text_only

# 便捷包装器（内部调用 run_generate_eval_advise）
python3 scripts/assistant/run_full_report.py \
  --metadata-csv datasets/CBIS_CLEAN_V2/metadata_clean.csv \
  --filter-view MLO --filter-density dense \
  --num-images 6 --tag-prefix round10 \
  --advisor-mode text_only
```

> **CSV 说明**：全流程与论文级复现推荐使用 **`datasets/CBIS_CLEAN_V2/metadata_clean.csv`**（训练图掩码外 burn-in 已清）。未清洗数据仍可用 **`datasets/CBIS_CLEAN/metadata_clean.csv`** 做对照。

> **训练数据集来源**：本项目训练数据来源说明为 [`CBIS-DDSM-DATASET`](https://github.com/sposso/CBIS-DDSM-DATASET)，Gradio 顶栏与 README 已同步展示该来源；仓库内 `CBIS_CLEAN_V2` 是在此基础上的清洗与结构化版本。

> **流水线 vs 手写生成**：`run_full_report` / `run_generate_eval_advise` 与 **`run_mammo_sd15` 默认一致**（**开启**频域后处理）。若加 **`--no-postprocess`** 则为 raw 对照，总耗时会略短、指标与观感与默认不同。

### 4.5 查看/回滚调参历史

```bash
# 查看历史
python3 scripts/assistant/run_full_report.py --show-history

# 回滚到第 2 条历史参数并生成
python3 scripts/assistant/run_full_report.py --rollback 2 \
  --metadata-csv datasets/CBIS_CLEAN_V2/metadata_clean.csv
```

## 5. 输出目录说明

**生成侧**：论文与 **Gradio「生成」** 写入 `outputs/generated/毕业论文_生成图像/<前缀>_<时间戳>_000/`。**`run_generate_eval_advise.py`（主入口）/ `run_full_report.py`（包装器）默认 `--output-base` 与此相同**；画廊会按批次内图片更新时间优先展示最新批次。

```
outputs/
├── generated/           生成图像批次目录（每批一个 *_000 子目录）
├── eval/                评审结果（summary.json + review_report.csv）
│   └── <批次名>/
│       ├── summary.json           汇总指标（pass_rate、违规率等）
│       ├── review_report.csv      每图详细评分
│       ├── top_<K>_images/        最优图复制目录
│       ├── advisor_qwen_text.md   # 可选：文本 API 分项输出（文件名历史遗留）
│       └── advisor_suggestions.md # 可选：合并 Markdown 分析报告
├── reports/
│   ├── LATEST_REPORT.txt          最新完整报告路径
│   ├── LATEST_NEXT_RUN.json       解析得到的下一批超参快照
│   ├── PARAM_HISTORY.json         最近 5 轮参数与指标历史
│   └── <tag>/FINAL_REPORT.md      每次一键流水线报告
└── lora/                LoRA 权重（训练产物）
```

## 6. 常见问题

**Q: 生成时提示 CUDA Out of Memory？**  
A: 减小 `--num-images`（每次 3 张）或改用快速预设。Gradio 固定全图模式并将保存长边控制在 1024，通常比旧 Patch 拼接更稳。

**Q: 找不到基座模型？**  
A: 检查 `hf_cache/sd15` 目录是否存在，或修改 `--base-model-local` 参数指向正确路径。

**Q: Gradio 无法访问？**  
A: 服务器环境下需开放对应端口（7860）；使用 `--host 0.0.0.0` 绑定所有网卡。云主机需在面板做 **端口映射** 或 Cursor **转发**，勿在未映射时在本机直接访问远端容器内的 `127.0.0.1`。

**Q: 日志提示 FastAPI 退出 / `Errno 98` 地址已在使用？**  
A: 多为 **8000 仍被旧 uvicorn 占用**。执行 **`bash apps/stop.sh`** 后重开，或用 **`lsof -iTCP:8000 -sTCP:LISTEN`** 查看占用进程；开发期避免同时对整仓开 **`uvicorn --reload`**，请用 **`bash apps/start.sh --api-reload`**（仅监视 `apps/`）。

**Q: 画廊很卡或一进画廊就卡住？**  
A: 确认已按上文使用 **缩略图 + 按需路径**；将无用批次 **`archive_legacy_generated_for_gallery.py`** 挪至 `_legacy`；可适当调低 **`GALLERY_PREVIEW_MAX`**。

**Q: 文本生成 API 返回 401 / 密钥无效？**  
A: 若使用 DeepSeek，检查 `DEEPSEEK_API_KEY`；若使用 Qwen/DashScope，检查 `QWEN_API_KEY` 与控制台额度。

**Q: 评审结果全部不通过（pass_rate = 0）？**  
A: 先确认是否使用了 `--eval-profile patch`（该模式下 stitch 检测更严）。常见违规：`TOO_UNIFORM`→降低 `--global-guide-blend`；`HIGH_BRISQUE`→确保后处理开启（`--postprocess`，默认即为开启），后处理可改善 BRISQUE 约 6-7 点（详见消融实验记录 `scripts/tests/run_ablation.py`）。`eval_profile=full` 下若仍全部不通过，检查 `--veto-group-min` 阈值与 `AREA_BAD`/`SHAPE_ODD`（两者通常与源图选择和组织区域大小有关，非单纯参数问题）。

**Q: `CBIS_CLEAN` 与 `CBIS_CLEAN_V2` 有什么区别？**  
A: **V2** 由 **`clean_training_labels.py`** 在乳腺掩膜外移除高亮连通域形式的 **burn‑in**。当前 Gradio 使用 V2 元数据与 v6 LoRA；复现旧实验可通过 CLI/API 手动改回旧 CSV 或旧权重。

## 7. 上线前自检

1. **依赖**：项目根 **`pip install -r requirements.txt`**；`bash apps/start.sh --help` 可验证 `fastapi/gradio` 可被拉起。  
2. **接线 smoke（无 GPU）**：

```bash
python3 scripts/tools/verify_ui_wiring.py
```

可加 **`--strict`**：缺 **V2 元数据** 或 **v4 LoRA final** 时报错退出（适合你方「必须答辩资源齐备」的机器）。  

3. **前后端是否「挂钩」**  
   - **Gradio** 标签页：**不通过 HTTP** 调本机 FastAPI，而是 **`subprocess`** 调用 `scripts/generation`、`scripts/evaluation`、`scripts/assistant`（与 **`apps/README.md`** 一致）。二者共享同一数据集与输出目录约定，互为并行入口。  
   - **评估** Tab：若在 **`datasets/jpeg/`** 下可见 **≥100** 张 `jpg/png`，界面会自动为 `review_generated_images` 附带 **`--real-images-dir`**，与 **`run_generate_eval_advise`** 行为一致，`summary.json` 可含 **`academic_metrics`**（FID 等）；不足 100 张则仅跑域内规则分项。  

4. **运行效率预期（经验值，依 GPU / 分辨率 / overlap 而变）**  
   | 动作 | 数量级 | 说明 |
   |------|--------|------|
   | 「生成」若干张，**精细**预设 · **全图** | ~数十秒～数分钟 | 单次整图 diffusion，无 patch 网格；答辩界面不展示 Patch 拼接 |
   | 「评估」full | ~数十秒～数分钟 | 与图数、`--eval-profile` 有关。 |
   | 「一键流水线」 | 生成 + 评 + （可选）顾问 API | **最慢路径**；下一轮参数见 **`outputs/reports/LATEST_NEXT_RUN.json`**，推荐 CLI **`run_full_report.py --from-latest-tuning`**；Gradio「调参历史」可 **`回滚`** 重跑 |

5. **`start.sh`** 自检：启动日志含 **`curl http://127.0.0.1:<端口>/ → HTTP 200`** 与 Gradio **`Running on`** 解析端口；若漂移端口以日志为准。
