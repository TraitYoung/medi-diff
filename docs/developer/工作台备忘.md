## 2. 和大模型分工（心法，不演戏）

Roughly：

- **对话里绑着仓库的那个模型**：改代码、跑命令、顺着堆栈/debug、读本文件。**具体路径 + 报错原文**比什么都重要。
- **`ask_advisor.py` / 流水线里的「文本后端」**：长文缩写、多角度 wording、从一大坨 log 抽 bullet——它不打开你的项目 tree，**别指望它单靠摘要改对具体行号逻辑**。
- 自己内心可以记成「一个写补丁，一个写作文」；真要一人包揽也行，只是一般更费 token、更容易飘。

会话里提高效率的习惯（任选，不必死板）：

- 一次只改一个问题；改完让一个最小命令验收（生成 1～4 张、或只跑 `review` 加 `--dry` 若能）。
- 贴错误时带上：**命令行 + 末尾 30 行 traceback**，以及相关文件路径。
- 「指标不对」这类话太泛；更好的是：**哪张图、哪个 `summary.json` 键、预期 vs 实际**。

## 3. 指标体系：自己脑子里怎么记

- **规则分项（含 BRISQUE、功率谱一类）**：钼靶体位/纹理/一票否决语义——**筛图和调主线参数**主要看这套。
- **`academic_metrics`（FID、KID、PRC、`sfid_spatial768`）**：Inception 族、和自然图像生成论文对齐——写「和某某工作可比」时再用；和规则总分 **不必单调**。

**2026-05-09 视觉优先记法**：总分高 ≠ 图好看。如果 B/E 组满分但 C/D/F 低，大概率是「统计数据对但肉眼塑料感」。权重已调整为：C(纹理)+D(伪影)+F(感知)=65%，B(灰度)+E(分布)=15%。调参时优先看三样：BRISQUE（<30 目标）、edge_density（≥0.005 底线）、VL 评分。

小样本次数不够时：**KID 可能跳过**；PRC 的 k 会被 **clamp 到 `min(N_gen, N_real) - 2` 一类上限**，别和手算假设拧着审。

分辨率：真图与生成分辨率不一致时，实现里会做 resize/crop 对齐——若你换了输入管线，第一反应检查 **预处理是否还跟评审脚本 assumption 一致**。

## 4. 常用入口（对照记忆）

**共享核心库**：`scripts/core/` — `PipelineConfig`（参数单一事实来源）、`SDPipeline`（可导入生成引擎）、`label_guard.py`（标签守护过滤器）、`postprocess_pipeline.py`（后处理纯函数）、`pipeline_orchestrator.py`（编排器）。`run_mammo_sd15.py` 和 `postprocess_freq.py` 均从此导入核心实现。

生成主线大致是：`run_mammo_sd15.py` + LoRA 路径（当前推荐 checkpoint 写在 `README` 目录树里）。

评审：`scripts/evaluation/review_generated_images.py`。关心论文指标就多确认传了 **`--real-images-dir`**（或等价），否则通用距离类可能缺席或不可靠。

全流程报告：`scripts/assistant/run_generate_eval_advise.py`（主入口）；`run_full_report.py` 为其子进程包装器。省钱默认 **`text_only`**；要图配文分析再 `both`。产物在 **`outputs/reports/`**，历史 eval 在下面 **`outputs/eval/`**。

Gradio/API 监听地址那些事：`README`/用户手册里写的是 **浏览器用 127.0.0.1**，监听 `0.0.0.0` 是部署语义——别混在一起记。

## 5. 容易忘的坑（个人清单，可往后加条目）

1. **`datasets/jpeg` 若在子目录里很深**：有的脚本只做**顶层计数**，会误判「0 张图」——遇到这种情况先看清脚本里到底是怎么 `glob` 的。
2. **极小 batch**：不要指望 4 张图就出稳的 KID/FID/PRC「论文级曲线」——脚本的 skip / 降级是预期行为。
3. **密钥**：只活在 `.env`；规则文件和 PR 里都别塞 key；患者可识别影像别喂给外包 API。
4. **归档脚本**：文件名或头上有 `[ARCHIVED]` / 归档说明的，写新实验就别当默认路径。
5. **视觉优先权重（2026-05-09）**：总分高但图难看 → 检查 C/D/F 组分数和 edge_density。VL 顾问的 `VL_AVG_SCORE` 比自动总分更接近人眼判断。`fast_iter.py` 默认启用 VL，用 `--no-vl` 关闭。
6. **BRISQUE 阈值演变**：05-09 收紧到 >45 → 05-14 放宽到 **>55**（配合其他阈值一起放宽，当前 CLI 默认）。真实钼靶 BRISQUE ~50-70。
