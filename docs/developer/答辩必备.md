## 四、核心架构（必背）

```
数据层: CBIS-DDSM → CBIS_CLEAN_V2 (metadata_clean.csv)
   ↓
训练层: SD1.5 UNet + LoRA (r=32, 全MLO视图)
   ↓
生成层: run_mammo_sd15.py (全图单次 img2img, 768→2048)
   ├── 源图筛选 (形态预筛: circ≥0.30, convex_defect≤0.45)
   ├── SD Pipeline (strength=0.42, guidance=6.8, DPM 50步)
   ├── Label Guard (启发式标签清除 + 亮斑修补)
   └── Postprocess (β校正 + CLAHE + Unsharp, blend=0.4)
   ↓
评估层: review_generated_images.py (16维 + hard_tags 一票否决)
   ↓
报告层: run_full_report.py → LLM 文本/VL 顾问 → LATEST_NEXT_RUN.json
```

## 五、关键选型论证

### 1. 为什么扩散模型不用 GAN？

- GAN 的模式崩溃（mode collapse）在医学图像不可接受——必须覆盖 fatty/scattered/heterogeneous/dense 全部密度
- 扩散模型渐进式去噪，生成质量更高、多样性更好

### 2. 为什么 SD1.5 不用 SDXL？

- SDXL 显存需求大（>12GB）、训练慢
- 更关键：SDXL 预训练数据缺乏乳腺 X 光纹理先验，不是"更大就更好"

### 3. 为什么 LoRA 不全量微调？

- SD1.5 UNet 860M 参数，全量微调需多卡
- LoRA 只训低秩矩阵（几百万参数），单卡可完成，且避免灾难性遗忘

### 4. 为什么全图 img2img 而非 patch 拼接？

- Patch 拼接存在接缝/banding 风险，每块独立扩散无法保证频率和亮度连续
- 全图单次推理天然无缝，答辩版默认模式

## 六、评估体系（答辩亮点）

**双轨并行，不混淆：**

| 轨道 | 用途 | 代表指标 |
|---|---|---|
| **域内规则**（主线）| 筛图 + 调参决策 | 16 维 A-F 组 + hard_tags 一票否决 |
| **通用深度指标**（辅线）| 横向对比其他论文 | FID / KID / PRC / sFID |

**A-F 六组（权重已调为视觉优先）：**

| 组 | 权重 | 内容 |
|---|---|---|
| A 构图 | 0.20 | 面积、圆度、胸肌、轮廓、垂直、解剖、裁剪 |
| B 灰度 | 0.10 | 统计、动态范围、极端值 |
| C 纹理 | 0.20 | RAPS、熵、梯度 |
| D 伪影 | 0.25 | 孤立圆、空洞、条带、圆密度、网格 |
| E 分布 | 0.05 | 直方图距离、密度类型 |
| F 频域 | 0.20 | 功率谱 β、BRISQUE |

**hard_tags（一票否决）**：BANDING、SHAPE_ODD、ARTIFACT_BUBBLES、CONTOUR_FRACTURED、GRID_SEAM、EDGE_VOIDS、BLOWOUT、AREA_BAD

> 代码中实际定义了 18 个 hard tags，以上 8 个是当前生成质量下的高频核心否决项。其余（DEAD_DARK、NO_PECTORAL、MODALITY_LOW、ANATOMY_NON_MAMMO、SKIN_LINE_MISSING、CENTER_OVEREXPOSED、CLIP_CROSS_ANATOMY、ARTIFACT_TRYPOPHOBIA、CLOSED_RING_ARTIFACT、MASK_AREA_OUT_OF_DIST）在 full-image 评估画像下部分降级为 soft penalty，不直接否决。

### 为什么这些指标是一票否决

判定原则：**不是"这张图不够好"，而是"这张图作为乳腺钼靶不可用"**。每个 hard tag 对应一种结构性/功能性失败，无法通过后处理或后期筛选挽回。

| 类别 | 标签 | 触发条件 | 为什么不能放行 |
|------|------|----------|----------------|
| **形状失败** | SHAPE_ODD | 乳腺圆度（circularity）超出硬阈值范围 | 圆度严重偏离意味着掩膜不是正常乳腺轮廓——可能是 img2img 过程中源图结构被扩散破坏，生成的"乳房"没有临床对应的解剖形状 |
| | AREA_BAD | 掩膜面积比（mask_ratio）完全超出 CBIS 真实分布范围 | 面积极度异常=生成物的尺度完全不合理，不是乳腺而是噪声块 |
| | CONTOUR_FRACTURED | 轮廓凸性缺陷过大（concavity / perim_ratio 超标） | 乳腺轮廓被"咬掉一块"或边缘锯齿状——这在对真实乳腺进行分割时不会出现，是生成失败的明确标志 |
| **伪影/生成失败** | BANDING | 傅里叶频谱中检测到定向条带（banding_score 低于阈值） | 周期性亮度条带不是乳腺纹理——是扩散模型在 latent 空间去噪时产生的结构性 artifact，在医学图像中会误导组织密度判读 |
| | GRID_SEAM | patch 拼接模式下 FFT 检测到网格峰值 | 网格接缝直接暴露了生成方法的工程痕迹——全图模式下已降级为 soft penalty |
| | ARTIFACT_BUBBLES | 孤立圆形亮连通域超过阈值 | 密集亮斑在真实钼靶中对应钙化簇或肿块边缘——大量假阳性亮斑会直接误导钙化检测等下游任务 |
| **曝光/灰度失败** | BLOWOUT | 过亮像素比例超过真实分布 p99 | 过曝区域的信息已丢失（饱和）——这些区域的纹理、钙化、肿块信号全部淹没，诊断可用性为零 |
| **边缘失败** | EDGE_VOIDS | 乳腺边缘黑色空洞区域超过阈值 | 边缘空洞意味着扩散过程在该区域没有"填充"乳腺组织——真实钼靶的乳房边缘是连续实质组织，空洞=明确的生成覆盖失败 |

> 关键区分：**BRISQUE 高不是 hard tag**——BRISQUE 衡量感知质量退化（模糊、噪声），但不代表结构不可用。一张稍模糊但形状正确、无伪影的生成图可以通过下游任务；一张有 BANDING 或 SHAPE_ODD 的图无论多清晰都不能用。

### 防挖坑标准话术

> "FID 改善不能等同于临床可用；BRISQUE 是感知质量参考不是医学语义指标；评估系统是生成图质量筛查，不等同于临床 accreditation。"

> "平均分用于批量趋势比较；最终展示样本需通过语义分层与人工复核。"

## 八、演示流程

1. **启动**：`bash apps/start.sh` → Gradio `localhost:7860`
2. **快速生成**：MLO + scattered，数量 4，点「生成」（约 20-30s）
3. **画廊**：切换 tab → 浏览全部（4 列预览）/ 源图对比（2 列并排）→ 精选导出（勾选→一键导出到 `outputs/curated/`）
4. **评估**：对刚生成的批次评分，展示通过率/均分/A-F 分项
5. **（可选）流水线**：`run_full_report.py` 完整跑一轮

备用方案：GPU 故障时用 `outputs/generated/` 下已有截图 + `outputs/eval/` 下已有报告。
