# 容错设计

> 本文档描述 medi-diff 系统在各环节的边界情况处理和异常降级策略，面向交接运维和答辩展示。

---

## 1. LLM 输出格式错误

### 问题

`ask_advisor.py` 调用外部文本 API（DeepSeek / GLM / Qwen），返回的 JSON 结构可能不符合预期——缺少字段、格式错误、或被注入了无关文本。

### 处理

**分层异常捕获**（`scripts/assistant/ask_advisor.py`）：

```python
def _extract_assistant_text(response: dict) -> str:
    try:
        return response["choices"][0]["message"]["content"]
    except (KeyError, IndexError, TypeError) as e:
        raise RuntimeError(f"Unexpected API response structure: {e}") from e
```

- 第一层：`KeyError` — 响应中缺少 `choices` / `message` / `content` 字段
- 第二层：`IndexError` — `choices` 数组为空
- 第三层：`TypeError` — 响应类型不匹配（如返回了字符串而非 dict）

所有异常统一转为 `RuntimeError` 向上抛出，由调用方的 `except RuntimeError` 兜底。

**参数解析回退**（`scripts/assistant/next_run_params.py`）：

```python
try:
    params = json.loads(extracted_json)
except (json.JSONDecodeError, TypeError, ValueError) as e:
    logger.warning("Failed to parse advisor JSON, using fallback: %s", e)
    return _default_params()
```

- 如果 LLM 返回的 JSON 无法解析，使用上轮参数作为回退，不中断流水线
- 解析成功但字段缺失时，用 `GenParams` 默认值填充

**CLI 入口兜底**（`main()` in `ask_advisor.py`）：

```python
try:
    result = text_advisor_banner_meta(...)
except RuntimeError as e:
    print(f"Advisor error: {e}", file=sys.stderr)
except urllib.error.HTTPError as e:
    print(f"HTTP {e.code}: {e.read()}", file=sys.stderr)
except Exception as e:
    print(f"Network/request error: {e}", file=sys.stderr)
```

三层兜底确保任何 API 异常都不会导致进程 crash，错误信息对调试友好。

## 2. API 限流与网络超时

### 问题

外部 API 可能因配额耗尽、网络抖动、服务端过载而返回 429/502/ timeout。

### 当前处理

- **超时控制**：文本 API 调用设置 60s 超时，Qwen-VL POST 校验每张图 60s 超时（`_post_timeout`）
- **明确报错**：连接失败时抛出包含状态码和响应体的异常信息，不会被静默吞掉
- **自动降级**：`text_advisor_banner_meta()` 在首选后端（DeepSeek）不可用时，自动尝试 Qwen fallback（通过 `resolved_text_backend()` 的重试逻辑）

### 改进方向（设计意图，待实现）

- 对 429 响应实现指数退避重试（初始等待 2s，每次翻倍，最多 3 次）
- 对 5xx 响应实现有限重试（最多 2 次）
- 重试耗尽后写入 `LATEST_NEXT_RUN.json` 标记 `eval_mode: fallback`，下一轮自动跳过顾问环节

## 3. 模型权重缺失

### 问题

SD1.5 base model 或 LoRA adapter 文件可能因磁盘故障、误删、路径配置错误而不可用。

### 处理

**启动时健康检查**（`apps/api_server.py`）：

```python
@app.get("/health")
def health():
    return {
        "sd15_model_exists": DEFAULT_MODEL.exists(),
        "sd15_lora_exists": DEFAULT_LORA.exists(),
        ...
    }
```

运维可在启动服务后立即 `curl /health` 确认关键文件就位，而非等到首次推理才发现。

**生成脚本的宽松处理**（`scripts/generation/run_mammo_sd15.py`）：

```python
if lora_path.exists():
    pipe.unet = PeftModel.from_pretrained(pipe.unet, str(lora_path))
else:
    logger.warning("LoRA path not found: %s", lora_path)
```

LoRA 缺失时仅 warn 并继续——允许仅用 base SD1.5 做调试，不会 crash。

**API Server 的 LoRA 选择逻辑**（`apps/api_server.py`）：

```python
_v4_final = ROOT / "outputs/lora/mammo_sd15_v4_clean/final_lora"
_v3_ck500 = ROOT / "outputs/lora/mammo_sd15_v3_retrain_20260502_1723/checkpoint-500"
DEFAULT_LORA = _v4_final if (_v4_final / "adapter_model.safetensors").is_file() else _v3_ck500
```

自动回退到备选权重，无需人工干预。

## 4. 源图像质量不足

### 问题

CBIS-DDSM 数据集中的 JPEG 图像质量参差不齐——部分图像存在极端宽高比、乳腺区域过小/过大、轮廓断裂、DICOM burn-in 标注残留。

### 处理

**多级预过滤**（`filter_source_pool()` in `scripts/generation/run_mammo_sd15.py`）：

| 过滤条件 | 阈值 | 动作 |
|---------|------|------|
| 宽高比超标 | `max(h,w)/min(h,w) > 2.2` | 丢弃 |
| 乳腺面积过小/过大 | `area_frac < 0.08` 或 `> 0.85` | 丢弃 |
| 圆形度不足 | `circularity < 0.30` | 丢弃 |
| 凸缺陷过大 | `convex_defect > 0.45` | 丢弃 |
| 背景标注伪影过多 | `_has_source_artifact_burden()` 综合评分 | 降级到备选池 |

当高质量源图不足时，自动从备选池补充（`logger.warning` 记录）：

```python
if len(good_entries) < num_images and artifact_entries:
    logger.warning("Artifact filter left only %d clean sources; using %d fallback sources", ...)
    good_entries.extend(artifact_entries[: num_images - len(good_entries)])
```

## 5. 生成结果异常

### 问题

即使源图质量合格，扩散模型仍可能产生伪影——条带（BANDING）、气泡（ARTIFACT_BUBBLES）、形状断裂（CONTOUR_FRACTURED）、过曝（OVEREXPOSED）。

### 处理

**评估系统硬否决**（`scripts/evaluation/review_generated_images.py`）：

16 维规则评分中，以下 `hard_tags` 触发一票否决（`ok = False`）：

- `BANDING` — 频域条带伪影
- `SHAPE_ODD` — 形态严重异常
- `ARTIFACT_BUBBLES` — 孤立圆形伪影过多
- `CONTOUR_FRACTURED` — 乳腺边缘断裂
- `EDGE_VOIDS` — 边缘空洞
- `GRID_SEAM` — Patch 拼接接缝
- `OVEREXPOSED` — 过曝
- `ARTIFACT_TRYPOPHOBIA` — 密集圆形伪影群

**病变样图案自动跳过**（`run_mammo_sd15.py` 生成后）：

```python
lesion_stats = _suspicious_lesion_stats(result, _main_tissue_mask(result))
if (lesion_stats["circumscribed_mass_count"] > 0
    or lesion_stats["calc_cluster_count"] > 0
    or lesion_stats["calc_dot_count"] >= 4):
    logger.warning("Skip candidate due to suspicious lesion-like pattern")
    continue  # 不保存，重抽下一张
```

这确保了生成图像不会意外包含类病灶结构（系统的设计目标是生成**正常**乳腺图像）。

## 6. 子进程超时与僵尸任务

### 问题

Gradio 和 FastAPI 通过 `subprocess.run()` 调用管线脚本，如果生成脚本因 GPU 死锁、OOM 等原因挂死，父进程会无限等待。

### 当前处理

- FastAPI 的 `JobRecord` 记录了 `started_at` / `finished_at` / `elapsed_seconds`，可用于事后排查挂死任务
- `start.sh` 中的看门狗进程（`_watch_proc`）监控 Gradio/FastAPI 主进程存活状态

### 改进方向（待实现）

- 对 `subprocess.run()` 加 `timeout` 参数（如 600s）
- 超时后 `subprocess.TimeoutExpired` → kill 子进程 + 写入错误日志
- 定期清理 `jobs` 字典中超过 1 小时的僵尸记录

## 7. Pydantic 输入校验

### 问题

API 调用方可能传入非法参数——数值越界、类型错误、路径注入。

### 处理

**FastAPI 请求体校验**（`apps/api_server.py`）：

```python
class GenerateSD15Request(BaseModel):
    num_images: int = Field(default=6, ge=1, le=50, description="生成张数")
    strength: float = Field(default=0.42, ge=0.05, le=0.80)
    guidance_scale: float = Field(default=8.5, ge=1.0, le=12.0)
    overlap_ratio: float = Field(default=0.90, ge=0.40, le=0.95)
    ...
```

非法参数在进入管线前被 FastAPI 拦截，返回 422 Unprocessable Entity + 明确的校验错误信息。

**路径遍历防护**（`_resolve_under_root()`）：

```python
def _resolve_under_root(value: str, *, must_exist: bool = False) -> Path:
    path = Path(value).resolve()
    try:
        path.relative_to(ROOT)
    except ValueError:
        raise HTTPException(400, f"路径必须位于项目目录内: {value}")
```

所有外部传入的文件路径强制在项目根目录内，防止目录遍历攻击。
