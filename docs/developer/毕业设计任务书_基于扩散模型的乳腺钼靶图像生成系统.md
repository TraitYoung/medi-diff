## 设计原始参数

**项目背景与应用场景：**

乳腺钼靶X线摄影是乳腺癌筛查的首选影像学方法，其图像质量的稳定性和数据多样性对提升AI辅助诊断模型的泛化能力与鲁棒性至关重要。然而，高质量、标注详尽且涵盖广泛解剖变异与病理类型的临床影像数据难以大规模获取，面临隐私、成本及类别不均衡等限制。传统的生成对抗网络（GAN）在生成高分辨率、细节丰富的医学图像时，常面临模式崩溃与训练不稳定的挑战。扩散模型作为新一代生成式AI的代表，通过渐进式去噪过程能生成质量更高、多样性更丰富的图像。本设计旨在构建一个基于扩散模型的乳腺钼靶图像生成系统，能够合成高度逼真且解剖结构合理的多样化乳腺影像，特别是针对特定解剖特征（如致密型乳腺）或病理模式（如早期微钙化），为下游诊断算法训练提供安全、可控的扩展数据源，具有显著的科研价值与临床应用潜力。

**预期功能：**

1. **数据预处理与特征分析模块：** 支持对标准乳腺影像数据集（如CBIS-DDSM、MINI-DDSM）的解析，实现图像标准化、ROI提取、基础数据增强及关键特征（如BI-RADS密度分级）的标签关联。
2. **可控图像生成核心模型：** 构建基于条件潜在扩散模型的深度生成网络，实现以文本描述（如"右侧乳腺，致密型组织，散在点状钙化"）或语义图为条件引导的高分辨率、高质量图像合成。
3. **模型训练与优化模块：** 实现扩散模型的高效训练流程，集成注意力机制与感知损失优化，并开发适用于医学图像的定制化评估指标。
4. **合成图像质量多维评估平台：** 提供定量评估（如FID、sFID、精度-召回率曲线）与可视化对比工具，支持合成图像与真实图像在像素级、特征级及放射科医生主观评分层面的综合比对。（与本条目对应的实现在仓库中以 **域内规则分项** 与 **Inception 族 FID/KID、PRC、sfid_spatial768** 等两套并行指标体系汇总，参阅 `docs/评价体系说明.md`。）
5. **可配置生成与简易交互界面：** 开发一个集成界面，允许用户通过下拉菜单、文本输入或草图勾画等方式配置生成条件，实时提交任务并浏览、筛选及导出生成的图像序列。

## 设计工作内容

1. **前沿技术调研与方案论证：** 深入研究扩散模型（DDPM， Stable Diffusion， LDM）及条件生成技术在医学影像领域的最新应用进展。对比分析扩散模型与GAN在医学图像生成任务上的性能、稳定性与计算成本，论证技术选型的合理性。完成详尽的文献综述与技术路线设计报告。

2. **数据集构建与预处理流水线开发：** 获取并管理公开乳腺钼靶数据集，进行脱敏与格式统一。设计并实现完整的预处理流水线，包括图像配准（如需）、强度归一化、异常值处理，以及结合临床报告的标签提取与结构化。构建条件-图像对用于模型训练。

3. **条件扩散模型的实现与训练：** 使用PyTorch框架搭建条件潜在扩散模型，设计并编码UNet去噪网络、条件编码器（如CLIP文本编码器或ResNet图像编码器）及调度器。实现并调试训练循环，集成混合损失函数（如噪声预测损失、感知损失），应用梯度裁剪、EMA等稳定训练的技术。在划分的训练集与验证集上进行多轮训练与迭代调优，保存中间检查点以供分析。

4. **端到端系统集成与部署：** 利用FastAPI开发高性能后端RESTful API，封装模型推理、图像后处理及评估计算功能。采用Streamlit或Gradio快速构建直观的前端交互界面，集成条件输入、任务队列、结果画廊与评估报告展示等功能模块。将数据预处理、模型服务、后处理流水线及用户界面无缝集成，确保系统可一键部署与运行。

5. **系统化评估与消融实验：** 设计全面的实验方案，在保留的测试集上定量评估生成图像的多样性与保真度。进行消融实验，分析不同条件注入方式、损失函数组件对生成效果的具体影响。组织小规模的专家评估（模拟），收集对合成图像临床合理性的定性反馈。

6. **设计文档编纂与成果总结：** 撰写系统架构设计文档、API接口文档及用户操作手册。严格按照规范完成毕业论文，重点论述模型创新点、实现难点、实验分析与结论。准备答辩陈述材料，制作展示系统核心流程、关键结果与对比分析的PPT及演示视频。

## 设计工作基本要求

- **系统性设计与实现：** 学生需完成从算法研究、模型训练到系统集成全流程的详细设计与实现，确保各模块高内聚、低耦合，代码结构清晰，工程实现规范。
- **技术先进性与可行性平衡：** 选题需紧跟生成式AI前沿，采用扩散模型等先进技术，但同时要充分考虑本科生知识储备与时间限制，设计方案应具备明确的可行性与阶段性目标。
- **论文撰写规范性：** 毕业论文必须严格遵守学校规定的格式与学术规范。内容需逻辑严密，数据翔实，图表清晰，能充分体现工作量、技术深度与个人贡献。
- **学术诚信与过程管理：** 恪守学术道德，所有引用需规范标注。定期与指导教师沟通进展，按时提交阶段成果，积极应对并解决项目中遇到的问题。
- **成果的完备性与可用性：** 最终应提交一个可运行的完整系统原型、所有源代码、训练好的模型权重（如许可允许）、详尽的文档及毕业论文。系统核心功能需经过充分验证。
- **鼓励探索与创新：** 在实现基础功能的前提下，鼓励在模型结构改进、条件生成方式、或评估方法上进行有益的探索与尝试，形成设计亮点。
- **代码与文档质量：** 代码应具有良好的可读性、注释和版本管理。所有文档应准确、完整、专业。
