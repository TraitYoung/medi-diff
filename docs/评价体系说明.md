# 评价体系说明

**读者对象**：医学影像、深度学习、医学图像生成与应用工程等相关研究与技术人员。

本文说明本系统在「合成钼靶图像质量」上采用的**两套互补指标**：二者量纲与假设不同，**不宜合并为单一总分**，而应分层次解读。

---

## 1. 为什么采用两套指标

| 关注点 | 典型读者关切 | 本仓库对应指标 |
|--------|----------------|----------------|
| **域内语义与成像合理性** | 图像是否具备钼靶体位/解剖与成像质量 proxies，是否与真实数据集统计一致（非临床 accreditation 替代） | **规则指标体系**（多维度分项、阈值、一票否决、`summary.json` 中通过率与维度均分） |
| **生成模型文献中的可比性** | 与扩散/GAN 等工作中常用的分布距离、覆盖度是否具有可比量级 | **通用深度特征指标**（FID/KID，及基于同一类 Inception 表征的 Precision–Recall、patches 空间 Fréchet 等），见下文 |

Inception 等骨干在 ImageNet 等自然图像上预训练，**对灰阶乳腺 X 光的语义并非最优表征**；因此通用指标适于**横向对比训练策略与批次**，**不能单独**等同于放射学意义上的「图像合格」。反之，规则指标偏重形态与统计 proxies，亦**不能保证**与传统论文中的 FID 单调一致。

---

## 2. 第一套：面向钼靶任务的规则与统计指标（主用）

### 2.1 范围

- **维度化分项**：构图与解剖（A）、灰阶（B）、纹理与频率（C）、伪影（D）、与真实分布偏差（E）、学术无参考分项（F）等；分项内为可计算标量与其到真实集基线的对照。
- **聚合**：分项映射为小分后与维度权重合成 **0–100 总分**（实现见 `scripts/evaluation/review_generated_images.py`）。
- **硬规则**：一票否决、`ok`/`tags`/`veto_*` 等，用于剔除明显不可用或语义风险较高的样本。
- **与真实库对齐**：在提供 `--real-images-dir`（或等价基线 JSON）时，阈值与校准分位可由真实钼靶抽样估计，以减少纯人工拍阈值带来的偏差。

### 2.2 文档索引

| 文档 | 内容 |
|------|------|
| `docs/评估标准说明.md` | 各分项含义、预处理与校准流程（篇幅较长） |
| `docs/乳腺钼靶生成图评价标准_v3.md` | 分层门禁、Top‑K、与 MQSA/ACR 概念的对应边界（不能直接当作临床 accreditation） |

## 3. 第二套：深度学习领域常用分布与流形指标（补充）

以下为批级汇总，写入 `outputs/eval/<批次>/summary.json` 中的 **`academic_metrics`**（键名与实现保持一致；部分项在样本过少时会跳过或由实现自动压低 `prc_neighborhood` 以保证数值稳定）。

| 名称 | 工具/实现要点 | 说明与局限 |
|------|----------------|------------|
| **FID** | `piq` + Inception 特征，`--real-images-dir` 且安装依赖 | Fréchet 距离，学界常用；**非**钼靶专用特征。 |
| **KID** | 同上，两组样本数量需对齐 | 小样本相对稳定；样本过少时脚本可能跳过。 |
| **Precision / Recall / F-score（PRC）** | `torch-fidelity`，Inception 嵌入上 KNN 流形意义上的 PR | 与传统二分类 PR 曲线**定义不同**；可扫多组邻域半径得到 `pr_curve`。 |
| **sfid_spatial768** | `pytorch-fid` Inception 中间块空间特征上 patch 分布的 Fréchet 距离 | 体现更细粒度空间统计；与文献中某一固定「sFID」实现**不一定逐比特等价**，适于**本项目内批次间相对比较**，使用前请阅读字段 `note` 中的定义说明。 |

**依赖**：详见根目录 `requirements.txt`（含 `piq`、`torch-fidelity`、`pytorch-fid` 等）；未安装时对应项为空或跳过并在日志中提示。

## 4. 已知局限与近期修正

### 4.1 2026-05-09 视觉优先权重重构

**问题**：生产批次自动化评分 91.2、通过率 100%，但 VL（Qwen VL）视觉顾问对 rank #1 图像仅给出 2/5 分。根因是评分系统以「统计分布匹配」为核心（灰度分布 B 0.18 + 真实偏差 E 0.08 = 26% 权重），而纹理自然度、边缘清晰度、伪影等视觉质量组（C+D+F）仅占 51%。且 BRISQUE 阈值过宽（>70 才违规），无法识别视觉上的过平滑/塑料感问题。

**修复内容**：

1. **维度权重重平衡**：`A=0.20, B=0.10, C=0.20, D=0.25, E=0.05, F=0.20`（原 `A=0.23, B=0.18, C=0.18, D=0.23, E=0.08, F=0.10`）。从灰度/分布向纹理+伪影+感知质量转移 ~15% 权重。

2. **BRISQUE 阈值收紧**：≤20 满分，>60 零分，>45 触发 HIGH_BRISQUE（原 ≤40 满分，>70 才违规）。真实 CBIS 钼靶 BRISQUE ≈ 50-70（mask-based），视觉可接受的生成图应明显优于此基线。

3. **edge_density 硬封顶**：当 `edge_density < 0.005`（过度平滑/模糊）时，C 组（纹理）分数上限 0.5。防止平滑图像通过其他纹理指标「搭便车」获得高分。

4. **VL 顾问权重提升**：视觉影响参数（`overlap_ratio`、`gabor_alpha`、`sharpen_strength`、`blend_sigma_divisor`）以 VL 建议为准（权重 0.6），文本指标分析降级为辅助参考。VL 顾问提示明确要求当视觉判断与指标冲突时以肉眼为准。

5. **tier1 门槛收紧**：`tier1_semantic_threshold` 0.75 → 0.80。

**重评效果**（生产批次 12 张）：总分 91.2→89.3，新检出 HIGH_BRISQUE 16.7%（2 张），edge_density 过低封顶 1 张。评分区分度从紧密聚类改善为 81.1-95.2 分布。

### 4.2 2026-05-08 评估系统可靠性修复

**GRID_SEAM 假阳性**：`patch_seam_check()` 在径向功率谱上的移动平均包络估计使用 `np.convolve(mode="same")` 做零填充，导致前 8 个频段残差异常大，而网格谐波目标频率恰好全部落入该区域，造成 100% 的系统性误报（包括真实钼靶原图也触发）。**已修复**（反射填充 + 1/f 低频区跳过 + 网格谐波只在可分辨频段检测）。修复后真实图与生成图均不再误报。

**软原因被硬否决**：`eval_profile="full"` 意图将 GRID_SEAM 降级为语义惩罚（-0.05 而非一票否决），但 tag 合并逻辑将 `soft_reasons` 与 `veto_reasons` 混入同一个被 `hard_tags` 检查的标签列表，导致降级无效。**已修复**（软原因只影响 `semantic_score`，不参与 `ok` 判定）。

### 4.3 BRISQUE 与 edge_density 的结构性 trade-off

2026-05-08 消融实验揭示：当前管线下 BRISQUE 与 edge_density 互为制约。后处理 beta 校正改善了 BRISQUE（均值 -6.9）但会压低 edge_density（全频段增益上限 3.0 修复后有所缓解）；关闭后处理 edge_density 升高但 BRISQUE 触顶（100.0）。突破此瓶颈需要**频段选择性后处理**（只衰减伪影频段，保留组织纹理频段）或**训练侧纹理增强**。

## 5. 使用建议（论文或技术报告）

1. **主结论**：优先报告规则体系的通过率、典型失败标签、以及与真实库的校准关系；图示建议采用 `review_report.csv` / `top_k_*` / 对比脚本产物。
2. **可比性**：在「与其它生成工作或消融对比」小节中并列报告 `academic_metrics`，并简述 Inception 域偏移及对钼靶解释的局限。
3. **不误读**：避免出现「FID 改善即等同于临床可用」或「总分高即可替代放射科审稿」等表述。
