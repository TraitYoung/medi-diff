# 乳腺钼靶生成图评价标准 v3

**读者**：医学影像、生成模型与安全筛选等相关研究与工程人员。

**目的**：在自动评审中区分「统计上与数据集接近」与「解剖/模态表达上可被推荐展示」，避免仅靠高分把明显语义异常图推入论文级 Top‑K。

**与指标体系总览**：本文件侧重 **域内分层与门禁语义**。**FID、KID、Precision–Recall、patch 空间 Fréchet（summary 中 `sfid_spatial768`）**等面向生成模型论文惯例的度量，归入 **另一套并行指标**，其定义边界与写法见 **[`评价体系说明.md`](评价体系说明.md)**——两套不设统一合成量纲，应分小节报告。

---

## 1. 依据来源

### 1.1 法规与临床质控依据

- FDA 的 MQSA（Mammography Quality Standards Act）要求乳腺摄影机构接受认证，认证机构会审查设备、人员、临床图像质量和实践流程。来源：<https://www.fda.gov/radiation-emitting-products/mammography-quality-standards-act-mqsa-and-mqsa-program>
- ACR Mammography Clinical Image Testing 要求提交 fatty 与 dense 两类病例，并由至少 2 位 ACR 放射科评审从 8 个方面评价临床图像：positioning、compression、exposure level、sharpness、contrast、noise、artifacts、exam identification。来源：<https://accreditationsupport.acr.org/support/solutions/articles/11000065937-clinical-image-testing-mammography-revised-05-02-2024->
- ACR Phantom Testing 用 ACR 认可 phantom 检查纤维、微钙化 specks、肿块等目标可见性，同时要求没有 clinically significant artifacts。来源：<https://accreditationsupport.acr.org/support/solutions/articles/11000065938-phantom-testing-mammography-revised-02-23-2026->
- ACR positioning improvement criteria 给出 2D mammography 摆位质量的 major/minor 条件，包括无运动、乳腺不可 cut-off、无其他身体部位遮挡、乳头 profile、PNL、CC/MLO 后方组织可见、IMF、胸大肌显示等。来源：<https://learningnetworksupport.acr.org/support/solutions/articles/11000115257-mammography-positioning-review-guidelines-table>

### 1.2 合成医学图像评估依据

近年的合成乳腺影像与医学生成数据论文常用三类验证：

- 感知真实性：放射科医生盲评 / Visual Turing Test、FID/KID、像素强度统计、SSIM/MS-SSIM 等。
- 诊断可用性：用合成数据训练或增强分类/检测模型，再在真实测试集上比较 ROC-AUC、敏感性、特异性、FROC 等。
- 隐私与多样性：患者重识别风险、membership inference、最近邻检索、类别/密度/视角覆盖度与 mode collapse。

参考示例：VinDr-Mammo synthetic mammography 研究使用 FID、癌症分类任务、患者识别模型与 membership inference 评估合成数据的保真、可用性和隐私。来源：<https://www.mdpi.com/2227-9709/12/4/112>

## 2. 本项目不能直接照搬的部分

本项目生成的是 PNG/JPEG 风格的 2D 图，不是完整 DICOM 采集流程，因此以下指标只能作为背景依据，不能直接作为自动评分主项：

- compression force、kVp、mAs、target/filter、dose 等采集参数：生成图没有真实采集参数。
- MTF、NPS/NNPS、DQE、phantom object visibility：需要 phantom 或原始探测器数据，生成结果本身无法可靠计算。
- ACR accreditation 的完整 clinical review：需要标准 4-view exam、DICOM 标签和人工评审。

可落地做法是：把 ACR/FDA 的“临床质量概念”转为图像级代理指标，并明确标注“本项目是生成图质量筛查，不等同于临床 accreditation”。

## 3. v3 评价原则

### 原则一：先判模态身份，再看统计质量

乳腺钼靶生成图的第一门槛不是 BRISQUE、FID 或功率谱，而是：

- 是否仍是 mammography / FFDM 风格图；
- 是否没有胸片、手足、牙科、超声、组织切片、拼贴 X 光等跨模态幻觉；
- 是否没有明显非乳腺器官或非乳腺结构。

如果此项失败，应直接 `ok=False` 或至少降到不可推荐层级。

### 原则二：Top-K 推荐采用“分层 + 精排”，不再只按总分

总分可以用于批量比较，但不能直接决定论文展示图。推荐顺序应为：

1. 先筛掉 hard veto。
2. 再按语义/模态风险分成 tier。
3. 只有 tier 1 可以进入 `top_k_recommended.txt`。
4. tier 2/3 只能进入 `top_k_candidates.txt`，必须带风险备注。

### 原则三：真实分布指标只能证明“像数据集”，不能证明“临床合理”

直方图、功率谱、梯度分布、密度类型等指标可以防止明显离群，但无法识别「统计分布正常、解剖语义异常」的样本。它们应放在语义门槛之后，用作质量精排，而不是主门槛。以 **FID/KID、PRC、`sfid_spatial768`** 为代表的 Inception 族指标（见 **`评价体系说明.md`**）同样只能说明在 **自然图像预训练特征空间**中的分布接近程度，**不可替代**上述解剖/模态门禁。

## 4. 推荐评价结构

### 4.1 第一层：硬门槛（Hard Veto）

以下任一触发，图像不得进入论文推荐集：

- 文件读取失败、非灰度医学图风格、尺寸异常。
- 明显非 mammography：胸片/手足/牙科/组织切片/超声/拼图式 X 光。
- 大面积过曝或死黑，影响乳腺主体判断。
- 乳腺主体 cut-off 严重，主体边界断裂，或出现多个不合理主体。
- 大量闭合圆环、蜂窝/鳞片纹理、局部 patch 拼贴、网格状重复。
- 其他身体部位或明显非乳腺器官遮挡乳腺主体。
- 与真实训练图高度近似，存在隐私泄漏风险。

### 4.2 第二层：语义与解剖分层

输出 `semantic_score` 和 `rank_tier`。

- tier 1：模态身份清楚，乳腺轮廓完整，纹理与解剖结构无明显冲突，无严重 artifact tag。可进入论文展示候选。
- tier 2：整体像钼靶，但有局部疑点，例如 BRISQUE 高、频谱异常、局部碎片、轻度过曝。只能作为候补，需人工复核。
- tier 3：语义可疑或局部结构明显不合理，即使总分高也不得推荐。
- tier 4：读取失败、hard veto 或总分/维度失败。

### 4.3 第三层：ACR 启发的图像质量项

把 ACR 的 8 项 clinical image review 转换为项目代理项：

- Positioning：主体是否完整；是否有合理的 CC/MLO 轮廓；MLO 是否可见胸大肌趋势；是否存在严重 cut-off。
- Compression：生成图无法直接判断压迫力，只用灰度均匀性、组织展开程度、异常折叠纹理作为弱代理。
- Exposure level：均值、动态范围、亮/暗区域比例，防止过曝或死黑。
- Sharpness：边缘/梯度统计、过度模糊检测。
- Contrast：局部对比度、CLAHE 后是否过强。
- Noise：真实噪声分布、BRISQUE/NIQE 类无参考质量指标；注意它们不是医学语义指标。
- Artifacts：banding、grid、patch seam、edge voids、圆形伪影、蜂窝纹理、文字/标记物异常。
- Exam identification：生成图不要求真实 DICOM 标识，论文图应避免伪造患者标识；若保留源图标记，需说明来自数据集或后处理未清除。

### 4.4 第四层：真实分布与技术统计

用于批量比较和同 tier 内精排：

- mask area ratio、circularity、contour concavity、perimeter ratio。
- intensity mean/std、dynamic range、bright/dark ratio。
- local entropy、gradient histogram distance、histogram Wasserstein。
- radial average power spectrum、ps slope beta、banding score。
- density type 与真实 CBIS 先验分布是否匹配。

### 4.5 第五层：合成数据专用验证

论文如果讨论“生成数据可用于增强/分析”，至少需要补充以下一种：

- Downstream utility：真实测试集上，比较 `real only`、`synthetic only`、`real + synthetic` 的分类/检测性能。
- Diversity：按乳腺密度、左右侧、CC/MLO、病灶/无病灶等统计覆盖度，检查 mode collapse。
- Privacy：最近邻检索、SSIM/LPIPS 近邻、embedding 近邻，防止生成图复制训练样本。
- Reader study：至少人工盲评少量图，记录“是否像钼靶”“是否可用于展示/训练”“主要失败原因”。

## 5. 当前代码对应关系

已有 `scripts/evaluation/review_generated_images.py` 已覆盖：

- 真实分布基线：histogram、RAPS、gradient、density prior。
- 形状与解剖：mask ratio、circularity、contour、pectoral、vertical artifact、anatomy chaos。
- 伪影：round artifacts、edge voids、banding、grid、BRISQUE、ps slope。
- 推荐分层：`rank_tier`、`semantic_score`、`final_rank_score`、`recommend_notes`。
- 输出拆分：`top_k_recommended.txt` 只写 tier 1；`top_k_candidates.txt` 写带风险候补；`top_k_by_total_score.txt` 保留旧总分榜。

当前短板：

- 没有专用 mammography modality classifier。
- CLIP 只能作为补充，且依赖本地模型缓存；没有缓存时会自动禁用。
- 皮肤线、乳头、PNL、IMF 等 ACR 摆位细节尚未形成稳定自动检测。
- 没有隐私近邻检测和 downstream utility 测试。

## 6. 推荐的落地顺序

### 立即落地

1. 保留当前严格推荐策略：没有 tier 1 就明确输出 `NO_TIER1_RECOMMENDATION`。
2. 对 Top-K 候补必须展示 `recommend_notes`，人工不能只看总分。
3. 对本项目论文展示图，要求：`ok=True`、`rank_tier=1`、`recommend_notes=pass`。

### 短期增强

1. 准备一个轻量 `P(mammography)` 分类器，作为 hard veto 或 tier 降级主信号。
2. 训练数据至少包含：真实 mammography 正样本、胸片、手足 X 光、牙科/颅骨 X 光、超声、自然灰度图、失败生成图负样本。
3. 分类器训练时统一裁剪/灰度/边框，避免模型只学黑边、水印、尺寸等伪特征。

### 中期增强

1. 做真实图最近邻检索，防止隐私泄漏和简单复制。
2. 做小规模人工读片表，记录失败类型，并把失败样本回灌到 modality classifier。
3. 若论文要声称生成图有数据增强价值，必须做 downstream classifier/detector 验证。

## 7. 论文表述建议

建议写法：

“本文不将 FID、BRISQUE 或灰度统计作为唯一质量依据。参考 MQSA/ACR 对乳腺摄影临床图像质量的要求，本文将生成图评价拆分为模态身份、解剖合理性、图像技术质量、真实分布一致性和合成数据可用性五个层次。对于出现跨模态幻觉、拼贴结构或严重解剖不合理的图像，即使其灰度/频谱统计接近真实数据，也不纳入展示或后续实验。”

不建议写法：

“平均分较高，说明生成图质量较好。”

必须改为：

“平均分用于批量趋势比较；最终展示样本需通过语义分层与人工复核。”
