## 一、当前未解决的问题

### 1. BANDING（条带伪影）— 间歇性，约 25% 违规

**现象**：生成图乳腺区内出现行/列方向亮度恒定的"死行"，评审 `banding_score < 0.70` 即判 BANDING（hard tag，直接否决 ok）。

**根因**：
- 全图模式下 BANDING 主要来自 SD1.5 模型本身对特定频率敏感的生成偏置，而非 patch 拼接引起
- 后处理 beta 校正（blend=0.4）和 CLAHE（clip=0.5）可能放大局部方差差异

**代码位置**：
- 条带检测：`scripts/evaluation/review_generated_images.py` → `compute_banding_score()`
- 评分：`grade_d3()`，阈值 `min_banding_score=0.7`

---

### 2. SHAPE_ODD / CONTOUR_FRACTURED — 形状不合规，约 25% 违规

**现象**：
- `circularity < 0.32` → SHAPE_ODD（hard tag，已从 0.45 放宽）
- `contour_concavity > 0.45` → CONTOUR_FRACTURED（hard tag，已从 0.35 放宽）

**根因**：
- 源图乳腺形态差异大（窄乳腺、极端角度 MLO）经 diffusion 后轮廓不规则
- 已新增源图形态预筛选（`filter_source_pool` 排除 circ<0.30 或 convex_defect>0.45 的源图），缓解但未根治
- 本质问题：SD1.5 的全图 img2img 对形状保持能力有限

**代码位置**：
- 源图筛选：`run_mammo_sd15.py` → `filter_source_pool()` 末部的"Shape quality pre-filter"
- 轮廓分析：`grade_a2()`（SHAPE_ODD），`grade_a4()`（CONTOUR_FRACTURED）

---

### 3. HIGH_BRISQUE — 感知质量偏低，约 25-50% 违规

**现象**：BRISQUE > 55 即判 HIGH_BRISQUE（非 hard tag，但拉低 F 组得分）。当前均值约 40，部分图达 55-60。

**根因**：
- 768px 推理结果 LANCZOS4 放大到 2048px 引入感知退化
- Unsharp mask（sharpen_strength=0.4）放大纹理噪声
- 注意：双边滤波（bilateral）在 2048px 上适得其反——会破坏乳腺高频功率谱并使 BRISQUE 恶化，**不建议启用**

**代码位置**：
- BRISQUE 评分：`grade_f2()`，阈值 brisque > 55 触发 HIGH_BRISQUE（已从 45 放宽）
- 后处理：`run_mammo_sd15.py` → 内联 `PostprocessParams(blend=0.4, sharpen_strength=0.4)`

---

### 4. bright_spots 伪影（legacy 指标，不直接否决）

**现象**：`bright_spots > 15`（乳腺区内 5-220px 高亮小连通域计数），阈值已从 10 放宽到 15。

**根因**：SD1.5 生成时在乳腺区注入小亮斑（来自预训练分布的残留模式）。

**已缓解**：`inpaint_interior_bright_spots()` 对孤立小亮斑做 TELEA 修补，`round_isolated` 从 8-10 降到 3-6。

---

### 5. Qwen-VL POST 验证未接入主路径

**现象**：`--no-qwen-vl` 参数被解析但从未在 `main()` 中使用；`ask_vl_label_bbox` 只存在于诊断脚本中。

**影响**：标签/文字残留依赖纯启发式清除（`erase_background_labels` + `erase_bright_border_labels`），无 VL 兜底。如需接入，参考 `scripts/tools/label_guard_diagnose.py` 中的调用方式。

### 3.2 生成流水线（全图模式，当前默认）

```
用户请求
  │
  ├─ Gradio ──subprocess──→ run_mammo_sd15.py --mode full-image
  ├─ FastAPI ──subprocess──→ run_mammo_sd15.py
  └─ CLI ──────────────────→ run_mammo_sd15.py

run_mammo_sd15.py 主循环（每张图）:

  1. 选源图（filter_source_pool）
     ├── 过滤体位/密度
     ├── 排除极端宽高比（> 2.2）
     ├── 形态预筛选（circ < 0.30 或 convex_defect > 0.45 → 跳过）
     ├── 质量排序（文件大小降序）
     └── source-seed 固定 / 时间戳随机

  2. 读取源图 → enhance_input_contrast（仅暗图 CLAHE）

  3. fullimage_generate（单次全图 img2img）
     ├── 缩放：长边 ≤ 768，短边 ≥ 384，硬上限 1024
     ├── DPM 20 步，strength=0.5，guidance=8.5
     └── 输出 LANCZOS4 放大到长边=2048

  4. Post-filter（legacy_label_guard=True）
     ├── erase_background_labels（启发式涂黑背景标签）
     ├── erase_bright_border_labels（边框条带小 CC 压暗）
     ├── feather_canvas_edge（feather_px=3）
     └── inpaint_interior_bright_spots（乳腺内小亮斑修补）

  5. _final_metal_sweep（金属 BB 圆检测 + TELEA 修补，最多 3 轮）

  6. postprocess（PostprocessParams：blend=0.4，sharpen_strength=0.4）
     ├── Winsorize（0.5%~99.5%）
     ├── β 校正（target_beta=2.8，blend=0.4）
     ├── CLAHE（clip=0.5，grid=8）
     └── Unsharp mask（radius=3，strength=0.4）

  7. 写 PNG + source_map.json + run_params.json
```

### 3.3 评审流水线

```
review_generated_images.py

  Stage 1: 漏斗
    veto_reasons → 写入 ok=False（GRID_SEAM 等）
    soft_reasons → 仅影响 semantic_score（penalty_map）

  Stage 2: 规则评分（16 维 → 6 组）
    A 组 构图 (0.20): 面积、圆度、胸肌、轮廓、垂直、解剖、裁剪
    B 组 灰度 (0.10): 统计、动态范围、极端值
    C 组 纹理 (0.20): RAPS、熵、梯度
    D 组 伪影 (0.25): 孤立圆、空洞、条带、圆密度、网格
    E 组 分布 (0.05): 直方图距离、密度类型
    F 组 频域 (0.20): 功率谱β、BRISQUE

  ok = (无 veto) AND (无 hard_tags) AND (各组分 ≥ 0.3) AND (total ≥ 50)
  hard_tags = {BANDING, SHAPE_ODD, EDGE_VOIDS, ARTIFACT_BUBBLES,
               CONTOUR_FRACTURED, GRID_SEAM, BLOWOUT, AREA_BAD, ...}
  注意：HIGH_BRISQUE 不在 hard_tags 中，不直接否决 ok
```

## 六、评估双重视图说明

- **`summary.json` → `thresholds`**：本轮实际使用的阈值（已校准或 CLI 硬阈值）
- **`summary.json` → `auto_calibrated`**：是否基于真实基线 percentiles 做了自动校准
- **`summary.json` → `per_image[].ok`**：最终判定（false 时 `tags` 字段列出触发的 hard_tags）
- **`outputs/reports/PARAM_HISTORY.json`**：最近 5 轮参数与指标历史
- **`run_params.json`**（在生成目录内）：完整记录本批次的 LoRA 路径、所有参数、seed 等，可完整复现

---

## 七、指标通过率 vs 目视质检

- **`pass_rate`** 来自规则分项与 `hard_tags`；当前严格模式（`--no-auto-calibrate`）下约 50%。
- **`HIGH_BRISQUE` 不在 `hard_tags`** 中，不直接否决 `ok`，只通过 F 组（权重 0.2）影响总分。
- **`GRID_SEAM`** 在全图模式下为频域假阳性风险；属 veto_reasons 而非 tags，仍导致 ok=False。建议对全图批次不带 `--enable-seam-check` 做一次对比评估以区分真伪接缝。
- **金属小球（BB marker）**：`_final_metal_sweep` 最多 3 轮 Hough + TELEA 修补（始终执行）。
- **文字与角标**：`_erase_text_residual_corners` **在仓库中不存在**（文档历史遗留）。标签清除依赖 `erase_background_labels` + `erase_bright_border_labels`。Qwen-VL bbox 未接入生成主路径。
- **自动校准**：`--real-images-dir` 应选用与 `--filter-density` 一致的真实钼靶 JPEG 集合，避免阈值向错误参考母体拟合。
